# Experiment 4: Forensic Timeline Generation

## Aim
To generate a chronological forensic timeline from filesystem MAC (Modified, Accessed, Changed) timestamps to reconstruct the sequence of events on a system.

## Theory
Every file on a filesystem maintains three timestamps:
- **M-time (Modification)** – last time file content was changed
- **A-time (Access)** – last time file was read
- **C-time (Change/Metadata)** – last time inode was changed (permissions, owner, etc.)

A forensic timeline correlates all these events chronologically to help investigators reconstruct what happened on a system and when.

## Requirements
- Python 3.x (standard library only)
- Linux or Windows system
- `mactime` (from The Sleuth Kit) – optional for TSK-based timelines
- `log2timeline` / `Plaso` – for advanced timeline generation

## Procedure

### Step 1: Run the script on a target directory
```bash
python3 generate_timeline.py /path/to/analyze
```
Default analyzes the current directory.

### Step 2: Review the output files
```bash
# View CSV timeline
cat timeline_output/timeline.csv | column -t -s,

# View anomalies in report
cat timeline_output/timeline_report.txt
```

### Step 3: Advanced – Using The Sleuth Kit (mactime)
```bash
# Generate bodyfile from disk image
fls -r -m / disk_image.dd > bodyfile.txt

# Convert to timeline
mactime -b bodyfile.txt -d > timeline.csv

# View sorted
sort timeline.csv | less
```

### Step 4: Advanced – Using Plaso (log2timeline)
```bash
pip install plaso
log2timeline.py --storage-file case.plaso /path/to/source
psort.py -o dynamic -w timeline.csv case.plaso
```
